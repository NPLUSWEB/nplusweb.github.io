webpack的使用

前端自动化的工具有很多：
第一种Gulp+Webpack构建前端自动化
第二种：Yeoman, Bower, Grunt构建前端自动化
第三种百度和腾讯也开发过.
在这里我研究的是Gulp+Webpack，如有兴趣可以研究其他的。

gulp主要做了下面几个工作：
1.css转化合并压缩
2.图片的雪碧图合并和base64
3.文件md5计算与替换
4.热启动，浏览器自动刷新
webpack解决了：
1.模块化开发
2.静态文件处理两大问题

基本步骤

第一步：安装node.js(https://nodejs.org/en/)
说明：gulp是基于node.js，所以必须安装node.js
打开cmd
node -v查看安装的nodejs版本，出现版本号，说明刚刚已正确安装nodejs
npm -v查看npm的版本号

第二步：创建一个文件存放项目：webpack
cd定位到目录，用法：cd + 路径

第三步：全局安装webpack
命令如下：npm install webpack -g
webpack 查看是否安装证明成功了

第三步：安装本地的webpack
命令如下：npm install webpack --save-dev

说明：我们全局安装了webpack，项目也安装了gulp

第四步：创建一个package.json文件

说明：创建一个package.json文件，这是一个标准的npm说明文件，里面蕴含了丰富的信息，包括当前项目的依赖模块，自定义的脚本任务等等。在终端中使用npm init命令可以自动创建这个package.json文件

第五步：创建一个webpack.config.js文件
说明：为了快速执行webpack

Loader:的使用
说明：Loaders是webpack中最让人激动人心的功能之一了。通过使用不同的loader，webpack通过调用外部的脚本或工具可以对各种各样的格式的文件进行处理，比如说分析JSON文件并把它转换为JavaScript文件，或者说把下一代的JS文件（ES6，ES7)转换为现代浏览器可以识别的JS文件。或者说对React的开发而言，合适的Loaders可以把React的JSX文件转换为JS文件。

CSS:

项目目录如下
说明：webpack提供两个工具处理样式表，css-loader 和 style-loader，二者处理的任务不同，css-loader使你能够使用类似@import 和 url(...)的方法实现 require()的功能,style-loader将所有的计算后的样式加入页面中，二者组合在一起使你能够把样式表嵌入webpack打包后的JS文件中。
